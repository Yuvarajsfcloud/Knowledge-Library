## What belongs here

Project documentation is distinct from the Knowledge Library (evergreen TOGAF/architecture theory) and Case Studies (retrospective post-mortems). A project page describes a **specific built system** — what it does, how it works, and how to operate it.

Write project docs when:
- A system has enough complexity that a new team member couldn't get started without documentation
- Architectural decisions were made that need to be preserved with their rationale
- The system will be handed off, extended by others, or revisited after a gap

Project docs are **living** — update them as the system evolves. They are not retrospective.
